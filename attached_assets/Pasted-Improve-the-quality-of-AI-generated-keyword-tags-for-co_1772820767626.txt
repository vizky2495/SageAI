Improve the quality of AI-generated keyword tags for content in the library. The current tags are too generic or too surface-level. Tags should be genuinely useful for a marketing manager searching for specific content for campaigns.

UPDATE THE TAG GENERATION PROMPT
When the AI analyzes uploaded content, use this specific prompt structure for generating tags:
You are a content analyst for a B2B marketing team. Read the 
following content and generate keyword tags that would help 
a marketing manager find this content when planning campaigns.

CONTENT TEXT:
[full content text]

CONTENT METADATA:
Product: [product]
Country: [country]
Industry: [industry]
Funnel Stage: [funnel_stage]

Generate 3 types of tags:

TOPIC TAGS (5-8 tags):
What specific subjects does this content cover? These should 
be the actual topics discussed, not generic categories.
- Good: "Year-End Payroll Close", "T4 Slip Generation", 
  "CRA Reporting Deadlines"
- Bad: "Payroll", "Tax", "Compliance"
The tag should be specific enough that someone searching 
for it would find exactly what they need.

AUDIENCE TAGS (2-3 tags):
Who specifically is this content written for?
- Good: "Accounting Firm Owners", "Canadian CPAs", 
  "Growing Practices 10-50 Employees"
- Bad: "Accountants", "Business Owners", "Professionals"
Include the seniority level, role, or business stage if 
the content targets a specific segment.

INTENT TAGS (2-3 tags):
What is someone trying to do or solve when they need 
this content?
- Good: "Modernize Practice Operations", 
  "Shift Compliance to Advisory", 
  "Reduce Year-End Processing Time"
- Bad: "Learn", "Improve", "Grow"
These describe the reader's goal or pain point, not 
the content's format.

Rules:
- Each tag must be 1-4 words
- Every tag must come from something actually discussed 
  in the content text. Never infer from title or metadata.
- Prefer specific over general: "Sage 50 Cloud Migration" 
  not "Cloud Migration"
- Prefer action-oriented over passive: "Automate Month-End 
  Close" not "Month-End Close"
- Include proper nouns when the content specifically 
  discusses them: "Will Farnell Methodology", 
  "CRA Digital Filing", "CPA Canada Standards"
- Do not repeat the product name as a standalone tag 
  unless the content specifically reviews or discusses 
  the product features
- If the content mentions specific regulations, standards, 
  or frameworks by name, include them as tags
- If the content mentions specific metrics or benchmarks, 
  include the concept: "30% YoY Growth", 
  "100% Cloud Adoption"

Return as JSON:
{
  "topic_tags": ["tag1", "tag2", ...],
  "audience_tags": ["tag1", "tag2", ...],
  "intent_tags": ["tag1", "tag2", ...]
}
