NEW FEATURE: AUTO-GENERATED KEYWORD TAGS
When the AI reads uploaded content, it must generate keyword tags — these are the key terms, concepts, and themes the content is talking about. These tags serve as a quick visual summary of what each content piece covers.
How to generate tags:

The AI reads the full content text and extracts 8-15 keyword tags per content piece
Tags should be specific and meaningful, not generic. Examples:

Good tags: "Year-End Payroll", "T4 Slips", "CRA Reporting", "Penalty Avoidance", "Canadian Tax Compliance", "Sage 50 Payroll Module", "Employee Deductions", "ROE Filing"
Bad tags: "Accounting", "Software", "Business", "Finance" — these are too generic to be useful


Tags should capture: specific topics covered, specific regulations or standards mentioned, specific product features discussed, specific industries or markets referenced, specific processes explained
Each tag should be 1-3 words maximum

Where tags appear:
On the comparison UI (on screen):

Below each content card, show the auto-generated tags as small colored pills
Content A tags in one color (e.g., teal pills)
Content B tags in a different color (e.g., jade pills)
Tags that appear in BOTH content pieces should be highlighted differently (e.g., green pills with a small "shared" indicator) — this instantly shows the user where content overlaps
Tags unique to one piece stay in their default color

In the PDF report:

Show tags for each content piece in the Content Overview section, directly below each summary
Format as small colored pills or bracketed labels: [Year-End Payroll] [T4 Slips] [CRA Reporting] [Penalty Avoidance]
Shared tags between both pieces marked with a "shared" label or different color
In the "What's Shared, What's Different" section, use the tags to visually show overlap: "Shared tags: [Payroll] [Canadian Market]. Unique to Content A: [CRA Reporting] [T4 Slips] [Penalty Avoidance]. Unique to Content B: [Sage 50 Setup] [Feature Overview] [Module Tour]"

In the Content Library cards (stored permanently):

After AI analysis, store the generated tags in the content_stored table as a new field: keyword_tags (JSON array)
Display these tags on library cards below the content summary
These tags persist permanently — they help users browse and understand content at a glance without opening the preview
Tags are searchable — when a user searches "CRA reporting" in the library, assets with that tag surface first
