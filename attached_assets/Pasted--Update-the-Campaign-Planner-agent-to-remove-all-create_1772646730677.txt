2. CONTENT UPLOAD & LIBRARY INTEGRATION
When a user uploads a content asset, the system must:
Step 1 — Store the asset in the campaign content library:

Save the file to the content library/database with all the metadata the user provided (asset name, content type, product, funnel stage, country, industry, date created)
Generate a unique asset ID that follows a human-readable naming convention: [Product]_[Country]_[Industry]_[ContentType]_[FunnelStage]_[ShortName] (e.g., "Intacct_US_NFP_PDF_TOFU_FundAccountingGuide")
Tag the asset as "New — Pending Performance Data" since it won't have any historical metrics yet
The asset should now appear in the content library for all future campaign plans and comparisons

Step 2 — Analyze the uploaded content:

If it's a PDF or document, extract the text content and analyze: topic coverage, key themes, CTA type, length, and structure
If it's an image or creative asset, note the format and let the user provide a description
Use this analysis to match it against the most relevant historical content in the library

Step 3 — Compare against historical performers:

Find the top 5 closest matches in the library by: content type + funnel stage + product + country + industry
Show the comparison table:

| Content Asset | Type | Stage | Product | Country | Industry | Leads | SQOs | Conv% | Engagement | Age | Performance Score | Source |

The uploaded asset will have no performance data yet, so its row should show "New — No historical data" in the metrics columns
The comparison is about content quality and relevance, not performance (since the new asset has none)
The agent should evaluate the uploaded content against top performers on: topic alignment, CTA strength, content depth, format effectiveness, and trend relevance (via web search)

Step 4 — Provide a verdict:

"This content is well-aligned with your top performers because [specific reasons]. Predicted performance range: [conservative/expected/optimistic based on comparable content]."
Or: "This content has gaps compared to your top performers. Specifically: [what's missing]. Recommend these changes before launching: [specific edits]."
Always frame this as optimization of the uploaded content, NOT as a recommendation to create something new.

4. CONTENT LIBRARY MANAGEMENT
Add a "Content Library" section accessible from the Campaign Planner main screen (alongside "New campaign plan" and campaign history). This section should:

Show all content assets in the library: both original dataset content and user-uploaded content
Allow filtering by: content type, product, funnel stage, country, industry, date range, performance score range
Show a status tag for each asset:

Active — has performance data, less than 6 months old
Aging — has performance data, 6-12 months old
Stale — has performance data, 12+ months old
New — recently uploaded, no performance data yet


Allow users to upload new assets directly from this view (same upload flow as the intake form)
Allow users to update metadata on existing assets (correct a wrong tag, update the funnel stage, etc.)
Show a library health summary at the top: total assets, assets by stage, content gaps by product/market, freshness distribution

5. REMOVE ALL "CREATE NEW CONTENT" REFERENCES
Search the entire agent codebase and prompts and remove:

Any option to "Create new content" or "Create new asset" from intake forms, system prompts, and UI
Any agent behavior that generates content briefs for net-new content (e.g., "Suggested title", "Key topics to cover", "Recommended format")
Any PDF section called "New Content Brief" or similar
Any references to "content creation" as a deliverable

However, keep these content-adjacent recommendations:

"Refresh this existing content by updating X, Y, Z" — this is optimization, not creation
"This content should be edited to include [trending topic]" — this is a revision suggestion
"Consider updating the CTA from [generic] to [specific]" — this is an improvement recommendation
"This content is outdated. Upload a replacement asset that addresses [current trends]" — this directs the user to bring in content, not to have the agent write it

The line is: the agent can tell you what's wrong with content and what should be different, but it does not write or outline new content. It evaluates and recommends.

7. AGENT SYSTEM PROMPT UPDATE
Add this to the Campaign Planner agent's system prompt:

"You are a content evaluation and campaign planning agent. You do NOT create new content. Your job is to:

Evaluate content that exists in the library or that the user uploads
Compare it against historically top-performing content using internal data
Check it against current industry trends using web search
Give a clear verdict: deploy as-is, refresh with specific changes, or recommend a better asset from the library
Plan the campaign deployment: which channels, what timeline, what KPIs to expect

When a user uploads new content, add it to the content library with the metadata they provide, then evaluate it against the closest historical matches. Never offer to write or outline new content. If content is fundamentally unsuitable, recommend a specific alternative from the library or tell the user what kind of replacement to source — but do not generate the replacement yourself.
Every data point must be tagged with its source: Internal Data, Web Research (with citation), Industry Benchmark, Calculated (with formula), or Assumption. No unattributed claims."
