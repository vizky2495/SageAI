## 3. Funnel Stage Logic

Classify each content record into **TOFU**, **MOFU**, or **BOFU** primarily using the `CONTENT` field (or another content ID/name field if explicitly indicated).

1. **Primary Rule (Keyword in Content ID/Name)**  
   - If the `CONTENT` (or a similar content ID/name column) contains:
     - `"TOFU"` ŌåÆ Funnel Stage = `"TOFU"` (Top of Funnel)
     - `"MOFU"` ŌåÆ Funnel Stage = `"MOFU"` (Middle of Funnel)
     - `"BOFU"` ŌåÆ Funnel Stage = `"BOFU"` (Bottom of Funnel)
   - Match is **case-insensitive** and can occur anywhere in the string.

2. **Tie Breaking / Missing Classification**
   - If multiple keywords appear, use this priority: **BOFU > MOFU > TOFU** (more bottom-funnel wins).
   - If no keyword is found and no explicit funnel-stage column exists:
     - Funnel Stage = `"UNKNOWN"`.
   - Still include UNKNOWN in counts where useful, but the primary dashboard focus is on TOFU / MOFU / BOFU.

3. **Extensibility**
   - If an explicit funnel stage column exists (e.g., `FUNNEL_STAGE`, `STAGE`, etc.), prefer that over the heuristic, but fall back to the heuristic when needed.

## 4. Key Metrics & Business Logic

For each stage (TOFU, MOFU, BOFU), compute key metrics. You must **map columns by meaning**, even if column names vary.

### 4.1 TOFU (Top of Funnel) ŌĆō Awareness & Early Engagement

**Goal:** Awareness and early engagement.

Look for columns that represent:

- **Engaged Sessions**  
  - Prefer columns with names like:  
    - `engaged_sessions`, `engaged_session`, `sessions_engaged`, or close variants.
  - If not available, use a general `sessions` or `visits` column as a proxy and clearly state this assumption in explanations.

- **New Contacts Generated**  
  - Prefer columns with names like:  
    - `new_contacts`, `contacts_created`, `new_leads`, `first_time_contacts`.
  - If only a binary flag per row exists (e.g., `is_new_contact`), sum that flag.

- **Visitor ŌåÆ Contact Conversion Rate**  
  - If you have both visitors and new contacts:  
    `visitor_to_contact_conversion = new_contacts / visitors`
  - If only sessions and new contacts:  
    `visitor_to_contact_conversion = new_contacts / engaged_sessions`  
    (or `/ sessions` when engaged sessions is missing).
  - Express as a percentage in narratives (e.g., 4.7%).

### 4.2 MOFU (Middle of Funnel) ŌĆō Lead Qualification & Consideration

**Goal:** Lead qualification and solution consideration.

Look for columns / logic describing **MQLs** (Marketing Qualified Leads):

- Treat a row as an **MQL** if any of the following is true:
  - A specific column like `is_MQL`, `MQL_flag`, or `MQL` equals 1 / true / "Yes" / "MQL".
  - A lifecycle or lead stage column (e.g., `lifecyclestage`, `lead_status`, `stage`) equals `"MQL"` or similar.
  - As a fallback, if a column indicates lead score above a certain threshold and is described as MQL logic, use that (and explain your assumption).

Metrics:

- **Primary KPI #1 ŌĆō MQLs**  
  - Count of distinct MQLs, or sum of MQL flag values.

- **Primary KPI #2 ŌĆō MQL Conversion Rate**  
  - If both new contacts and MQLs exist:  
    `MQL_conversion_rate = MQLs / new_contacts`
  - If leads exist instead of contacts:  
    `MQL_conversion_rate = MQLs / total_leads`
  - Use the denominator that is most semantically consistent; mention which denominator you used when explaining.

- **Quality KPI ŌĆō Lead Score / MQL Quality Index**  
  - If a lead score column exists (e.g., `lead_score`, `FORM_SCORE1`, or similar):
    - Compute average score for MQLs.
    - Optionally compute percentile breakdown (top 10%, median, bottom 10%).
