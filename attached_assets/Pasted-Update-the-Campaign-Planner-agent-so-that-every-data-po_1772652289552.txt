SOURCE CLASSIFICATION SYSTEM
Every data point must be tagged with one of these source types:

[Internal Data] — Comes directly from the dataset via the Data Agent. This includes: lead counts, SQO counts, conversion rates, engagement times, content performance metrics, channel breakdowns, asset-level statistics. These are facts from your own data. Display with high confidence.
[Web Research] — Comes from web search during plan generation. This includes: industry trends, market sizing, regulatory updates, competitor activity, current best practices, content format trends. These should include the specific source name and year (e.g., "[Web Research: Blackbaud 2024 Nonprofit Technology Report]", "[Web Research: HubSpot State of Marketing 2025]"). If a web source cannot be named specifically, tag it as "[Web Research: Industry benchmark]" and note it should be independently verified.
[Industry Benchmark] — Standard B2B marketing benchmarks that are commonly accepted but not from a single source. This includes: average email open rates, LinkedIn CTR ranges, typical B2B CPL ranges. Tag as "[Industry Benchmark]" and note the general source category (e.g., "[Industry Benchmark: B2B SaaS average per HubSpot/Demand Gen Report]").
[Calculated] — Derived by the agent from internal data or a combination of sources. For example, if the agent multiplies an internal conversion rate by an estimated audience size to project leads, tag it as "[Calculated: based on Internal Data conversion rate × estimated audience]". Always show the calculation logic briefly.
[Assumption] — Not backed by any data source. Used when the agent fills a gap with a reasonable professional estimate. Tag as "[Assumption]" and briefly state the basis (e.g., "[Assumption: based on comparable B2B SaaS campaigns in similar verticals]"). These should be flagged visually so the user knows to validate them.

RULES FOR SOURCE INTEGRITY

Never present an assumption as internal data. If the dataset doesn't contain a specific metric, do not imply it does. Tag it honestly.
Never invent web sources. If the agent cites a report (e.g., "Blackbaud 2024"), it must have actually found this via web search. If the web search didn't return a specific source, tag the claim as "[Industry Benchmark]" or "[Assumption]" — never fabricate a citation.
Flag confidence level for projections. When the agent projects performance (e.g., "Expected 850-1,100 downloads"), show the calculation chain:

Base metric: 331 avg downloads per Intacct TOFU asset [Internal Data]
Multiplier: 3x for new vertical launch [Assumption: based on comparable vertical launches in dataset]
Result: 993 midpoint, range 850-1,100 [Calculated]

This lets the user evaluate whether the projection methodology is reasonable.
Web research claims must be current. Do not cite reports older than 18 months unless no newer data exists. If citing an older source, note the year and add: "(latest available; verify for updates)."
Internal data claims must be traceable. When referencing internal data, specify enough detail that the user could verify it in the dataset. Instead of "Internal Data" alone, prefer "Internal Data: 47 Intacct TOFU assets, avg across all channels" or "Internal Data: US financial services segment, 2024-2025."
Separate facts from recommendations. Data points are sourced. Recommendations are the agent's strategic judgment based on those data points. The recommendation itself doesn't need a source tag, but every data point supporting it does. Example:

"LinkedIn is recommended as the primary channel." ← This is a recommendation, no source tag needed.
"LinkedIn drove 38% of Intacct TOFU leads (Internal Data) and nonprofit finance leaders show 61% weekly active usage on the platform (Web Research: LinkedIn B2B Report 2025)." ← These are the supporting facts, both tagged.
