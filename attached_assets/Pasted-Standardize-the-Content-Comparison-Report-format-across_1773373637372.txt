STANDARDIZE: REPORT STRUCTURE
For 2-piece comparisons — target 5-6 pages:
Page 1: Cover — title with short readable names, date, author, Sage branding
Page 2: Asset Identity table + Content Overview (concise summary per piece, 3-4 sentences each) + Tags grouped by type + Metadata issue flags
Page 3: Key Topics (1-2 sentence explanation per topic per piece, not full paragraphs) + Comparison Insight (one paragraph connecting the two)
Page 4: Audience Resonance table + Shared vs Different (bullet points, not paragraphs)
Page 5: Could Be Improved (2-3 bullet points per piece, only for content actually read) + Verdict (one paragraph) + Suggestions (maximum 4, 1-2 sentences each)
Performance data as inline text where relevant, not a separate page unless both pieces have meaningful data.
For multi-piece comparisons (3-5 pieces) — target 5-7 pages:
Page 1: Cover — title listing all content short names, date, author, Sage branding
Page 2: Rankings + Verdict — ranking table with scores (methodology noted), "Best for" callout pills (Best for Leads, Best for Engagement, Best for Conversion), AI Verdict (one paragraph). This is the executive summary page — a busy stakeholder should be able to read only this page and get the key takeaway.
Page 3: Asset Identity table (all pieces as columns) + Content Overview (compact — 2-3 sentences per piece) + Tags per piece
Page 4: Audience Resonance table (all pieces as columns) + Key Topics per piece (rendered correctly — no "undefined")
Page 5: Shared Themes + Differentiators + Content Gaps (all as bullet points)
Page 6: Could Be Improved (per piece, bullet points) + Suggestions (maximum 5, concise)
Performance data as inline text where relevant.

FEATURES UNIQUE TO MULTI-PIECE COMPARISONS (keep these):
These elements only appear in multi-piece comparisons (3+ pieces), not in 2-piece comparisons:

Scoring and ranking table — numbered ranking with scores out of 100, one-line reason, methodology footnote
"Best for" callout pills — Best for Leads, Best for Engagement, Best for Conversion. Only show when there's enough engagement data to support the claim. If no engagement data, skip these callouts.
Content Gaps section — issues that are common across the entire set (e.g., "all three lack CTAs"). This is more valuable in multi-piece comparisons because it reveals patterns across a content set.
Shared Themes section — what the content set has in common. Useful for understanding whether a set of content tells a coherent story or has redundancy.


PAGE COUNT RULES:

2-piece comparison: maximum 6 pages including cover
3-piece comparison: maximum 7 pages including cover
4-5 piece comparison: maximum 8 pages including cover
Adding more content pieces should NOT linearly increase page count. Use tables and compact layouts to keep it tight.
Never generate a page with just a header and no content — skip the section or merge it with another section.
Sections with less than 3 lines of content should be merged with an adjacent section, not given their own page.
Dynamic page count: if there's genuinely less to say (e.g., one piece wasn't readable), reduce the page count accordingly. A 2-piece comparison where one piece wasn't readable should be 3-4 pages, not 6 empty ones.
